Welcome to the **nestjs-durable** documentation.

`@dudousxd/nestjs-durable` brings durable execution to NestJS. You write a workflow as ordinary
async code — call a step, use its result, call the next — and the engine records every step's
output. If the process crashes or you deploy mid-run, it resumes from the last checkpoint instead
of starting over. Steps can run locally in NestJS or on a remote worker (even a Python one), but
it stays one workflow, with one source of truth and one timeline.

Head to the [Getting Started](/docs/getting-started) guide to run your first workflow in a few
minutes, or use the sidebar to explore the concepts, transports, stores and observability.

## What you get

- **Crash-proof by replay.** Each step is checkpointed. On a crash or deploy the workflow re-runs
  from the top, but completed steps replay their saved result — so they run exactly once and only
  unfinished work executes.
- **Steps across apps and languages.** A typed remote step is dispatched over a pluggable
  transport — to another process or a Python worker on the same queue. The workflow code never
  changes; only the transport does.
- **Durable sleep and signals.** Pause for minutes or months with `ctx.sleep` (no compute while
  waiting), or wait on a human approval / webhook with `ctx.waitForSignal`. Both survive restarts.
- **Bring your ORM.** State lives in Postgres through a `StateStore` interface, with MikroORM and
  TypeORM adapters and auto-schema on boot.
- **See the whole flow.** A built-in control plane renders each run as a graph; OpenTelemetry and
  a Telescope watcher give you two more views of the same event log.

## The shape of it

```ts
@Workflow({ name: 'checkout', version: '1' })
export class CheckoutWorkflow {
  async run(ctx: WorkflowCtx, order: Order) {
    await ctx.step('reserveStock', () => this.inventory.reserve(order));
    const charge = await ctx.call(chargeCard, { orderId: order.id, amountCents: order.total });
    const approval = await ctx.waitForSignal<{ approved: boolean }>(`approve:${order.id}`);
    if (!approval.approved) return { status: 'rejected', chargeId: charge.chargeId };
    await ctx.step('ship', () => this.carrier.dispatch(order));
    return { status: 'shipped', chargeId: charge.chargeId };
  }
}
```

If the process dies after `chargeCard`, the run resumes on boot and `reserveStock` + `chargeCard`
are **not** re-run — their results come straight from the checkpoints.

## Packages

| Package | Role |
| --- | --- |
| `@dudousxd/nestjs-durable-core` | Engine, replay, sleep/signals, `remoteStep`, interfaces |
| `@dudousxd/nestjs-durable` | NestJS module, `@Workflow`/`@DurableStep`, recovery, auto-schema |
| `@dudousxd/nestjs-durable-transport-event-emitter` | In-process transport (zero-infra default) |
| `@dudousxd/nestjs-durable-transport-bullmq` | BullMQ/Redis transport for cross-process steps |
| `@dudousxd/nestjs-durable-store-mikro-orm` · `-store-typeorm` | ORM state stores |
| `@dudousxd/nestjs-durable-dashboard` | Embedded control-plane SPA |
| `@dudousxd/nestjs-durable-otel` · `-telescope` | OpenTelemetry & Telescope integrations |
| `durable-worker` (PyPI) | Python worker SDK |
